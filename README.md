# fib_api

## アプリケーションの概要
本プロジェクトは、指定されたn番目のフィボナッチ数を返すREST APIです。
開発言語はPython、フレームワークはFastAPIを使用し、稼働環境としてRenderを利用しています。
フィボナッチ数列の仕様に基づき、リクエストされた順番（n）に対応する正確な数値をJSON形式で返却します。

## ソースコードの構成
* main.py: APIのエントリポイントです。リクエストの受付、入力値のバリデーション、エラーハンドリングを担当します。
* fib_calc.py: フィボナッチ数の計算ロジックを分離したファイルです。
* unit_test.py: 計算関数の正常系および異常系を検証するための自作テストスクリプトです。
* requirements.txt: 動作に必要なPythonライブラリの一覧です。

## 処理の概要

### main.py の動作
リクエストを受け取った際、以下の順で処理を行います。
1. サーバー内部で予期せぬエラーが発生した場合は、一律で500エラーを返却するようにハンドリングしています。
2. FastAPIの機能により、引数 n が整数以外（文字列や小数など）の場合は自動的に400エラーを返却します。
3. n が1未満の場合、およびサーバー負荷軽減のため10000を超える場合は、400エラーを返して処理を中断します。
4. 事前検証を通過したものだけ fib_calc.py の計算関数に渡し、結果を返却します。

### fib_calc.py の動作
関数単体での安全性を確保するため、内部で再度 n の型が厳密に整数であるかを検証します。
計算処理は以下のロジックで行っています。
1. n が1の場合は 1 を返します。
2. n が2以上の場合は、変数 a=0 と b=1 を用意します。
3. 直前の2つの項を足し合わせる（result = a + b）ループ処理を繰り返し、変数の値を更新していくことでn番目の数値を求めます。

## エラーレスポンスの仕様
不適切なリクエスト時には、以下の形式でレスポンスを返却します。

```json
{"status": 400,"message": "エラーに応じたメッセージ"}
```

発生条件とメッセージの対応は以下の通りです。
* サーバー内部エラー発生時 (status: 500): サーバー内部でエラーが発生しました。
* nが1未満の場合 (status: 400): nは1以上の整数を指定してください。
* nが10000を超える場合 (status: 400): nは10000以下にしてください。
* nが整数でない場合 (status: 400): nには整数を指定してください。
* 計算側で想定外の入力を検知した場合 (status: 400): 不正な入力です。

## ユニットテストの実行方法
リポジトリのルートディレクトリで以下のコマンドを実行してください。

```bash
python3 unit_test.py
```

実行後、標準出力に各テストケースの入力値、出力結果、期待値、および合否が出力されます。
